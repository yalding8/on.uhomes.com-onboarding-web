# UAT 测试指南 — uhomes 供应商入驻平台

> **版本:** 1.0 | **日期:** 2026-02-26
>
> **生产环境地址:** https://on-uhomes-com-onboarding-web.vercel.app

本指南包含 uhomes 供应商入驻平台的逐步测试场景，按用户角色分为两部分。

**开始前准备：** 请确保你能正常访问工作邮箱收件箱，用于接收 OTP 验证码。

## 第一部分：BD（商务拓展）测试流程

**测试账号：** 使用你的 `@uhomes.com` 邮箱地址。

登录后会自动跳转到管理后台。

### BD-01：OTP 验证码登录

| 步骤 | 操作                                                       | 预期结果                                                                     |
| ---- | ---------------------------------------------------------- | ---------------------------------------------------------------------------- |
| 1    | 打开 https://on-uhomes-com-onboarding-web.vercel.app/login | 看到 "Welcome to uhomes" 标题、邮箱输入框和 "Continue with Email" 按钮       |
| 2    | 不输入邮箱，直接点击 **Continue with Email**               | 显示错误提示："Please enter a valid email address"                           |
| 3    | 输入你的 `@uhomes.com` 邮箱，点击 **Continue with Email**  | 页面切换到验证码步骤，显示 "We've sent an 8-digit secure code to [你的邮箱]" |
| 4    | 查看邮箱收件箱                                             | 收到一封包含 8 位数字验证码的邮件                                            |
| 5    | 只输入 4 位数字                                            | **Secure Login** 按钮保持禁用（灰色不可点击）                                |
| 6    | 输入完整 8 位验证码，点击 **Secure Login**                 | 按钮文字变为 "Verified"，随后跳转到 `/admin/suppliers`                       |

### BD-05：编辑草稿合同

> 此测试需要一个合同状态为 **DRAFT** 的供应商。如果没有，请让管理员先审批一个供应商申请。

| 步骤 | 操作                                              | 预期结果                                                                                                                                                                    |
| ---- | ------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1    | 在供应商详情页，找到状态为 **DRAFT** 的合同       | Action 列显示 "Edit Contract" 链接                                                                                                                                          |
| 2    | 点击 **Edit Contract**                            | 跳转到合同编辑页面，看到 9 个表单字段、**Save** 和 **Push for Review** 按钮                                                                                                 |
| 3    | 检查 9 个字段是否存在且可编辑                     | Partner Company Name、Partner Contact Name、Partner Address、Partner City、Partner Country、Commission Rate (%)、Contract Start Date、Contract End Date、Covered Properties |
| 4    | 检查自动预填                                      | Partner Company Name 和 Partner City 可能已从供应商申请信息中自动填充                                                                                                       |
| 5    | 将 **Partner Contact Name** 修改为 "Test Contact" | 字段正常接受输入                                                                                                                                                            |
| 6    | 将 **Commission Rate** 设为 `15`                  | 字段接受数字输入                                                                                                                                                            |
| 7    | 将 **Contract Start Date** 设为今天的日期         | 日期选择器正常工作                                                                                                                                                          |
| 8    | 将 **Contract End Date** 设为一年后的日期         | 日期选择器正常工作                                                                                                                                                          |
| 9    | 点击 **Save**                                     | 按钮显示 "Saving..."，然后出现成功提示："Contract fields saved"。字段值保持不变                                                                                             |
| 10   | 点击 **Push for Review**                          | 先保存，然后显示 "Contract pushed for review"。Save 和 Push 按钮消失（合同不再是 DRAFT 状态）                                                                               |
| 11   | 点击 **Back to Supplier** 返回                    | 合同表格中该合同状态变为 **PENDING_REVIEW**                                                                                                                                 |

## 状态流转参考

```
申请状态：   PENDING（待审核）──→  CONVERTED（已转化）或 REJECTED（已拒绝）
                                      │
合同状态：              DRAFT（草稿）──→ PENDING_REVIEW（待审核）──→ CONFIRMED（已确认）──→ SENT（已发送）──→ SIGNED（已签署）
                                                                                                              │
房源状态：                                                        extracting（提取中）──→ incomplete（待完善）──→ previewable（可预览）──→ ready_to_publish（待发布）──→ published（已发布）
```
