请求参数配置

测试流程

1 点击登录跳转

用户点击「跳转到 Uhomes 登录」按钮，请求：
GET /buser/api/oauth/goto/jobs?redirect=https://jobs.uhomes.com/

2 Uhomes 登录

在 Uhomes 登录页面输入账号密码，完成登录

登录成功后，系统会生成 tokens 并跳转回调地址

3 接收回调

登录成功后，系统会重定向到 Jobs 站点的回调地址，URL 中会携带：uhouzz_token=xxx 和 access_token=yyy

4 获取用户信息

使用 access_token 调用以下接口获取用户信息：

  • GET /api/oauth/getUserInfo
  • POST /api/oauth/refreshToken (刷新 token)
5 完成

Jobs 站点接收用户信息后，创建/关联本地账号，完成登录流程

快速测试

快速测试流程：

  1. 点击「跳转到 Uhomes 登录」按钮
  2. 在新标签页打开登录页面，输入账号密码登录
  3. 查看回调 URL 中是否有 uhouzz_token 和 access_token
  4. 复制 access_token，在浏览器控制台运行以下测试命令：

1. 获取用户信息测试：

curl -X GET "https://www.uhomes.com/buser/api/oauth/getUserInfo" \
  -H "X-Client-Appid: 1009" \
  -H "access_token=YOUR_ACCESS_TOKEN"

2. 刷新 Token 测试：

curl -X POST "https://www.uhomes.com/buser/api/oauth/refreshToken" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -H "X-Client-Appid: 1009" \
  -H "access_token=YOUR_ACCESS_TOKEN"

错误码参考：

  • 1001: access_token 不能为空
  • 1002: token 无效或已过期
  • 1003: 用户不存在
  • 5000: 服务器错误
